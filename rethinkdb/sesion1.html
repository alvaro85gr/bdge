2. Descripción de la base de datos¶

2.1 ¿Qué es RethinkDB?¶

RethinkDB es una base de datos escalable JSON construida para su uso en aplicaciones web en tiempo real. Invierte la arquitectura tradicional de base de datos con un nuevo modo de acceso. En lugar de solicitar los datos con el cambio, el desarrollador puede pedirle a RethinkDB que continuamente le mande los datos actualizados resultantes de una consulta a una aplicacin en tiempo real. Esta arquitectura permite reducir drsticamente el tiempo y esfuerzo necesario para construir aplicaciones escalables en tiempo real.

Adems de ser diseada desde cero para este tipo de aplicaciones, RethinkDB ofrece un lenguaje de consultas flexible (ReQL), operaciones inutuitivas y una API de monitorizacin. Con todo, es fcil de configurar y aprender a usar. As pues, teniendo en cuenta su arquitectura, los principales casos de uso de RethinkDB pueden ser:

  • Aplicaciones web y mviles colaborativas
  • Plicaciones de anlisis en tiempo real
  • Juegos multijugador
  • Tiendas en tiempo real
  • Dispositivos conectados

2.2 ¿Cómo funciona RethinkDB?¶

El acceso y manipulacin de los datos es sencillo. Tanto desde la interfaz web como desde APIs para Ruby, Python, Java, y JavaScript/Node.js, podemos lanzar consultas al servidor de RethinkDB. En la web se puede encontrar un sencillo tutorial de apenas 10 minutos en el que ponemos en marcha la base de datos y realizamos algunas consultas sencillas Por ejemplo, para la API de Python podemos consultar este

2.3 Posibilidades de modelado de datos¶

Al ser una base de datos JSON, permite todas las posibilidades que un documento JSON nos ofrece: documentos formados por campos y valores de los mismos, cuyos valores pueden ser a su vez otros documentos, listas, etc.... Cada documento y cada tabla poseen una clave primaria primaria que los identifica unvocamente. Si no se especifica, se crea automaticamente un campo id y se le asigna un valor cuando se introduce en la base de datos.

2.4 Características¶

Transacciones atómicas¶

Las operaciones que afectan a un solo documento se realizan de forma atmica, salvo si se trata de una operacin no determinista. Adems, la actualizacin de mltiples documentos no se hace de forma atmica.

Framework de procesamiento map/reduce¶

RethinkDB permite el uso de map-reduce de forma sencilla. La secuencia de la operacin tiene la siguiente forma:

  • Una operacin group opcional que particiona los elementos en multiples grupos.
  • Una operacin map que filtra y/o transforma los elementros de la secuencia (o del grupo) en una nueva secuencia (o grupos de secuencias).
  • Una operacin reduce que agrega los valores producidos por el map en un solo valor (o un solo valor por cada grupo).

Replicación multiservidor¶

RethinkDB te permite fragmentar y replicar el clster en base a las tablas. Los ajustes se pueden controlar fcilmente desde la consola de administracin web. Adems, los comandos de ReQL para la configuracin de tablas permiten la capacidad de creacin de scripts y un control ms preciso sobre la replicacin, distribuyendo rplicas para tablas individuales en grupos de servidores definidos por el usuario utilizando etiquetas de servidor.

RQ4 Reciprocity in StackOverflow¶

Explicacin:

  1. Borramos de la tabla posts aquellos posts sin autor (da problemas luego en la consulta)
  2. Unimos para cada respuesta con su pregunta (procurando que el autor de la respuesta y la pregunta no sean el mismo) y nos quedamos con el campo que indica el autor de la pregunta y la respuesta.
  3. Con distinct, tomamos solo los pares nicos.
  4. Luego reordenamos los campos para que aparezca primero el id ms pequeo. Esto har que los pares preguntador-respondedor caigan en la misma "clave"
  5. Agrupamos por clave compuesta, y contamos el nmero de casos de cada clave
  6. La solucin sern aquellos pares cuya cuenta sea 2 (pues existia el par como preguntador-contestador y contestador-preguntador)
